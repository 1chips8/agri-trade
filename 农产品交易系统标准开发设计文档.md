## 2. 项目概述

项目名称：

```text
基于 Spring Boot + Vue 的农产品直销交易与订单履约系统
```

项目目标：

```text
构建一个支持农产品商家入驻、商品发布审核、消费者浏览下单、模拟支付、订单履约、评价、通知和后台统计的交易平台。
```

第一版边界：

```text
不接真实支付
不接真实物流
不采用微服务
不接 Elasticsearch
不实现复杂售后
不实现 Kubernetes 部署
```

第一版重点：

```text
核心业务闭环
数据库设计
接口设计
权限与状态控制
Redis 缓存与并发控制
RabbitMQ 延迟任务与异步通知
Docker Compose 部署
```

## 7. 核心业务流程

### 7.1 商家入驻

```text
提交申请
-> merchant_apply 状态 PENDING
-> 管理员审核
-> 通过：merchant_apply 状态 APPROVED，新增 merchant，用户角色改为 MERCHANT
-> 拒绝：merchant_apply 状态 REJECTED，记录 reject_reason
-> 生成通知
```

### 7.2 商品发布审核

```text
商家发布商品
-> product.audit_status = PENDING
-> product.sale_status = OFF_SALE
-> 管理员审核
-> 通过：audit_status = APPROVED
-> 拒绝：audit_status = REJECTED
-> 审核通过后商家可上架
```

### 7.3 下单

```text
选择购物车项
-> 校验商品和库存
-> Redis 防重复提交
-> Redis 商品库存锁
-> 按商家分组生成订单
-> 扣库存
-> 创建订单主表
-> 创建订单明细
-> 创建支付记录
-> 写库存流水
-> 清理购物车
-> 发送订单超时取消消息
```

### 7.4 支付与履约

```text
PENDING_PAYMENT
-> 用户模拟支付
-> WAIT_SHIPMENT
-> 商家发货
-> SHIPPED
-> 用户确认收货
-> COMPLETED
```

### 7.5 超时取消

```text
订单创建后发送延迟消息
-> 30 分钟后消费
-> 如果仍为 PENDING_PAYMENT
-> 取消订单
-> 返还库存
-> 关闭支付记录
-> 写库存流水
-> 生成通知
```

## 11. Redis 设计

Key：

```text
login:token:{token}
product:hot:list
product:detail:{productId}
order:submit:lock:{userId}
product:stock:lock:{productId}
admin:statistics:overview
```

用途：

```text
登录状态
商品缓存
防重复提交
库存分布式锁
统计短期缓存
```

订单提交：

```text
使用 order:submit:lock:{userId} 防止重复提交。
使用 product:stock:lock:{productId} 控制并发扣库存。
```

缓存更新策略：

```text
商品修改、审核、上下架时删除 product:detail:{productId}
热门商品缓存过期后重新加载
```

## 12. RabbitMQ 设计

队列：

```text
order.delay.cancel.queue
notice.create.queue
```

交换机：

```text
order.exchange
notice.exchange
```

路由键：

```text
order.cancel.delay
notice.create
```

订单超时取消：

```text
订单创建成功后发送延迟消息。
30 分钟后消费者检查订单状态。
如果仍为 PENDING_PAYMENT，则取消订单、返还库存、关闭支付记录、写库存流水并生成通知。
```

异步通知：

```text
支付成功、审核完成、订单发货、订单取消等事件通过 MQ 异步写入 message_notice。
```

幂等：

```text
订单取消消费者必须先判断订单状态。
非 PENDING_PAYMENT 状态直接忽略。
```
